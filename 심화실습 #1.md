## 3. 코드 리뷰 결과

### 3.1 AI가 생성한 코드를 얼마나 이해할 수 있었는지?

- 이해도: 0점으로 시작했으나, 심화실습으로 인해 1점으로 상승했습니다.
- 생성형AI가 생성한 코드는 프로그램의 전체 틀을 먼저 갖춘 후 세부 기능을 설계했습니다. 시스템 제목, 실행할 기능의 틀을 갖춘 후 기능 실행 방법에 대한 코드 설계가 이어졌습니다. 그리고 사용자의 입력에 따른 출력값 설정, 잘못된 입력값에 대한 출력값 등을 설정했습니다.

### 3.2 최초에 입력한 프롬프트와 그 결과물에 대한 평가

#### 최초 프롬프트

```
[넌 어떤 인공지능보다 똑똑한 최고성능의 인공지능이고] [난 프로그래밍 강의 처음 듣는 입문자]
[과제로 생성형AI에게 프롬프트를 제시해서 일정관리 프로그램 코드를 생성해주면 내가 그걸 읽으면서 코드 해독의 중요성을 깨닫는걸 받았어]
[C언어로 일정 관리 프로그램을 만들어줘 기본 기능: 1.일정 추가(제목,날짜,시간,설명), 2.일정 조회, 3.일정 수정 4. 일정 삭제
고급기능: 1. 키워드로 일정 검색 2. 날짜별 정렬 3. 파일 저장/불러오기 기능을 넣어 동작하게 해줘.]
```

#### 최종 코드 및 실행 결과

> 본 문서 가장 아래 `부록` 섹션에서 `최종 결과물 코드`와 `실행 결과`에 각각 코드 및 실행 결과를 복사 & 붙여넣기로 첨부합니다.

### 3.3 최종 결과물을 얻기까지의 경험

  1. 생성형AI에게 입력할 프롬프트 정교한 설계, 생성형AI와 프로그래머의 수준과 역할을 지정하고 원하는 출력물을 정확하게 입력하여 출력값을 얻어낼 수 있었습니다.
  2. 생성형AI와 출력한 결과값을 코드스페이스에서 시행착오를 겪으며 수정을 거쳐나갔습니다.
  3. 한 단계씩 수정을 거치며 텍스트 형식의 프로그램을 생성할 수 있었습니다.

## 4. 결론
바이브코딩 시 원하는 출력값을 만들기 위해서 입력단계에서 정교한 설계가 필요하다는 것을 배웠습니다. 일정 관리 프로그램이라는 단어에 자연스럽게 그래픽 기반의 프로그램 생성을 생각한 후 생성형AI에게 코드를 생성하라고 지시했으나, 생성형 AI는 텍스트 기반의 프로그램을 생성해냈습니다. 생성된 프로그램의 코드 분석 과정에서 단순하게 코드 선언부만 있고 구현부가 없는 함수 생성으로 링킹 에러가 발생할 수 있다는 사실에 대해서 알 수 있었습니다. 이를 수정하는 과정에서도 저장과 재컴파일을 진행하지 않으면 해결한 에러가 지속해서 재발할 수 있다는 사실 또한 알 수 있었습니다.

### 최종 결과물 코드

```
- AI가 생성한 전체 코드 첨부
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

#define MAX_SCHEDULES 100
#define FILE_NAME "schedules.dat"

typedef struct {
    char title[50];
    char date[11];  // YYYY-MM-DD
    char time[6];   // HH:MM
    char desc[100];
} Schedule;

Schedule list[MAX_SCHEDULES];
int count = 0;

void addSchedule();
void viewSchedules();
//void updateSchedule();
//void deleteSchedule();
void searchSchedule();
void sortSchedules();
void saveToFile();
void loadFromFile();

int main() {
    loadFromFile(); // 시작 시 파일 불러오기
    int choice;

    while(1) {
        printf("\n--- 2026 갓생 관리 시스템 ---\n");
        printf("1.추가 2.조회 3.수정 4.삭제 5.검색 6.정렬 7.종료\n");
        printf("선택: ");
        scanf("%d", &choice);
        getchar(); // 버퍼 비우기

        switch(choice) {
            case 1: addSchedule(); break;
            case 2: viewSchedules(); break;
            //case 3: updateSchedule(); break;
            //case 4: deleteSchedule(); break;
            case 5: searchSchedule(); break;
            case 6: sortSchedules(); break;
            case 7: saveToFile(); return 0;
            default: printf("잘못된 입력입니다.\n");
        }
    }
}

void addSchedule() {
    if (count >= MAX_SCHEDULES) return;
    printf("제목: "); gets(list[count].title);
    printf("날짜(YYYY-MM-DD): "); gets(list[count].date);
    printf("시간(HH:MM): "); gets(list[count].time);
    printf("설명: "); gets(list[count].desc);
    count++;
    printf("추가 완료!\n");
}

void viewSchedules() {
    for(int i=0; i<count; i++) {
        printf("[%d] %s | %s %s | %s\n", i+1, list[i].date, list[i].time, list[i].title, list[i].desc);
    }
}


void searchSchedule() {
    char keyword[50];
    printf("검색어 입력: "); gets(keyword);
    for(int i=0; i<count; i++) {
        if(strstr(list[i].title, keyword)) {
            printf("찾음: %s (%s)\n", list[i].title, list[i].date);
        }
    }
}

void sortSchedules() {
    Schedule temp;
    for(int i=0; i<count-1; i++) {
        for(int j=0; j<count-i-1; j++) {
            if(strcmp(list[j].date, list[j+1].date) > 0) {
                temp = list[j];
                list[j] = list[j+1];
                list[j+1] = temp;
            }
        }
    }
    printf("날짜순으로 정렬되었습니다.\n");
}

void saveToFile() {
    FILE *fp = fopen(FILE_NAME, "wb");
    fwrite(&count, sizeof(int), 1, fp);
    fwrite(list, sizeof(Schedule), count, fp);
    fclose(fp);
}

void loadFromFile() {
    FILE *fp = fopen(FILE_NAME, "rb");
    if(fp == NULL) return;
    fread(&count, sizeof(int), 1, fp);
    fread(list, sizeof(Schedule), count, fp);
    fclose(fp);
}
